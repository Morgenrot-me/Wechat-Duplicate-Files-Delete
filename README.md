# 删除微信重复文件的简易工具

## 功能说明

本工具用于识别并删除微信类重复文件，但只会处理满足以下全部条件的普通文件：

1. 文件名在扩展名前以 `(数字)` 结尾，例如 `a(1).jpg`、`a(12).txt`
2. 同目录下存在对应原文件，例如 `a.jpg`、`a.txt`
3. 原文件与尾号文件大小一致
4. 删除前会进行 SHA-256 哈希校验，只删除内容完全一致的文件

扫描阶段只比较文件大小（速度快），删除前会逐个进行哈希校验确保安全。
如果用户选择的是 `Weixin Text`、`xwechat_files` 或账号目录，程序会自动只定位到实际有用的 `msg/file` 目录，不扫描无关加密目录。
全目录树扫描时界面会持续刷新当前进度，避免长时间看起来无响应。

只有满足以上条件的文件才会被移动到回收站，不会直接永久删除。

## 使用方法

### 直接运行

```bash
python main.py
```

启动程序时会先弹出一次使用说明，明确说明候选规则、校验方式和回收站保护。

启动后：

1. 点击”扫描（递归）”或”扫描（当前目录）”按钮
2. 选择要检查的目录（可直接选 `Weixin Text`、`xwechat_files`、某个账号目录，或直接选 `msg/file`）
3. 扫描完成后，界面会显示所有发现的重复文件列表
4. 每个文件默认勾选，可以取消勾选不想删除的文件
5. 点击”删除选中项”按钮，将勾选的文件移入回收站

执行完成后，程序会在界面中显示选中文件数、成功删除数和失败数。

## 验证规则

运行轻量验证脚本：

```bash
python verify.py
```

当前验证覆盖：

- 尾部编号匹配
- 多位编号匹配
- 非尾部编号不匹配
- 原文件不存在时不删除
- 大小不同不删除
- 哈希不同不删除
- 目录名带 `(1)` 不删除
- 符号链接候选不删除
- 扫描异常时不中断整体流程
- 递归模式能检查子目录

## 打包说明

运行 `build.bat` 生成可执行程序，输出一般位于 `dist/` 目录。

`build.bat` 不再依赖当前命令行里的 Conda Python，而是固定使用 `py -3.11` 检查项目内的 `.venv-build` 虚拟环境，并在其中使用以下打包依赖：

- `PyQt5`
- `send2trash`
- `PyInstaller`

脚本执行顺序如下：

1. 检查 `py -3.11` 是否可用
2. 检查 `.venv-build\Scripts\python.exe` 是否存在
3. 检查 `.venv-build` 中是否已安装打包依赖
4. 运行 `verify.py`
5. 调用 PyInstaller 打包

`build.bat` 当前不会自动创建虚拟环境，也不会自动安装依赖。缺少 Python 3.11、缺少 `.venv-build` 或缺少依赖时，脚本会直接提示并停止执行。

## 注意事项

- 当前版本运行时只使用 `main.py` 中的界面定义。
- 本工具不会因为文件名相似就直接删除，必须通过原文件存在和哈希一致校验。
- 打包为窗口程序后，结果反馈通过 GUI 展示，不依赖控制台输出。
