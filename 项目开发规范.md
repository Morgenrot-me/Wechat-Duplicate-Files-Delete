# 项目开发规范

## 目录作用
当前项目为小型单文件桌面工具项目，所有核心逻辑集中在根目录，不强行拆分前端、后端、数据库结构。

## 文件列表
- `main.py`：桌面工具主程序。
- `ui.ui`：历史界面设计稿，不参与当前运行时加载。
- `build.bat`：Windows 本地打包脚本。
- `README.md`：使用与打包说明。
- `claude.md`：本项目本地协作规范。
- `根目录说明.md`：根目录文件说明。
- `verify.py`：本地轻量验证脚本。

## 技术说明
1. 使用 PyQt5 构建界面。
2. 使用 `send2trash` 将文件送入回收站，避免永久删除。
3. 使用文件名尾部编号规则与 SHA-256 哈希校验识别重复文件。
4. 当前 UI 直接定义在 Python 文件中，不通过 `.ui` 文件动态加载。

## 使用示例
1. 运行 `python main.py` 启动工具。
2. 点击“是”执行递归删除。
3. 点击“否”执行当前目录删除。
4. 运行 `python verify.py` 执行本地规则验证。

## 扩展方式
1. 如需恢复 Qt Designer 工作流，应先统一 UI 单一真源，再决定是否改为动态加载 `ui.ui`。
2. 如需增加更多重复文件规则，必须先补验证用例，再扩展匹配逻辑。

## 注意事项
1. 不得因为文件名相似而直接删除，必须通过哈希校验确认。
2. 不得删除目录。
3. 修改代码后必须同步更新 README 与根目录说明文档。
