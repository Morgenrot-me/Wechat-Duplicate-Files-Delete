# 根目录说明

## 目录作用

根目录存放当前桌面工具的全部源代码、打包脚本、说明文档与验证脚本。

## 文件列表

- `main.py`：主程序入口，包含界面定义、目录选择、重复文件扫描、预览列表、单文件选择与删除功能。
- `ui.ui`：历史界面设计稿，当前版本不由程序运行时加载，仅供参考。
- `build.bat`：Windows 打包脚本，会先检查 `py -3.11` 和项目内 `.venv-build` 是否可用，运行验证脚本，再调用 PyInstaller 生成可执行文件。
- `README.md`：面向使用者的说明文档，描述运行方式、删除规则和打包步骤。
- `claude.md`：本项目本地协作规范与安全边界说明。
- `项目开发规范.md`：当前项目的开发规则说明。
- `verify.py`：轻量验证脚本，用于验证尾部编号匹配、原文件存在检查、哈希一致性、扫描异常容错、符号链接排除和目录保护等关键规则。
- `claude.md` 同时承担本地 CLAUDE 规范文件角色，Windows 下文件名大小写不区分。

## 技术说明

1. 项目基于 PyQt5 构建桌面界面，使用 QListWidget 显示候选文件列表。
2. 通过 `send2trash` 将目标文件移入回收站。
3. 通过尾部编号模式匹配候选文件，再通过原文件存在、大小一致和 SHA-256 哈希一致确认重复文件。
4. 当前 UI 代码内嵌在 `main.py` 中，因此该文件是界面与逻辑的唯一真源。
5. 用户可以在扫描后预览所有候选文件，并通过复选框选择要删除的文件。
6. 全目录树扫描会分批刷新进度到 GUI，避免长时间无响应。
7. 程序启动时会先弹出使用说明，明确删除规则、校验方式和回收站保护。
8. 扫描入口会自动把 `Weixin Text`、`xwechat_files` 或账号目录收敛到实际需要的 `msg/file`，避免扫描无关目录。

## 使用示例

```python
python main.py
python verify.py
```

```bat
build.bat
```

## 扩展方式

1. 新增文件时，必须先在本说明文档补充文件用途。
2. 如需增加新删除策略，必须先补验证脚本场景，再接入 GUI。
3. 如需引入新的构建或测试工具，必须先更新本目录说明和 README。

## 注意事项

1. 当前 `ui.ui` 不是运行时加载文件，修改它不会直接改变程序行为。
2. 删除逻辑必须始终遵守“候选规则 + 哈希校验”。
3. 打包模式为窗口程序，错误反馈必须可在 GUI 中看到。
